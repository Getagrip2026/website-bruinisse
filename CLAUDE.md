# CLAUDE.md

## Project Context

This project is a website for parents about the exploration of a
potential collaboration school between CBS Op Dreef and OBS De Meerpaal
in Bruinisse.

The purpose of the website is: - to inform - to create understanding -
to support trust - to build connection between parents

IMPORTANT: There is NO final decision yet. This is an exploration phase.

------------------------------------------------------------------------

## Core Principles (MANDATORY)

Claude must always follow these principles:

### 1. Neutrality (CRITICAL)

-   Do not favor one school over the other
-   Always represent CBS Op Dreef and OBS De Meerpaal equally
-   Do not suggest that a decision has already been made
-   Avoid any wording that implies certainty about outcomes

### 2. Clarity

-   Use simple, structured layouts
-   Use clear headings and sections
-   Avoid complex or abstract explanations

### 3. Accessibility

-   Write at B1 language level
-   Use short sentences
-   Avoid jargon and policy language

### 4. Trust & Transparency

-   Clearly distinguish:
    -   what is known
    -   what is being explored
    -   what is undecided

------------------------------------------------------------------------

## Design Guidelines

Claude must strictly follow the design guidelines defined in:

brand_assets/brand-guidelines.md

Key requirements: - Use light blue as primary color - Use green only as
accent - Use lots of whitespace - Use rounded corners and soft shadows -
Avoid visual clutter - Keep layout calm and structured

DO NOT: - create a flashy or marketing-style design - overuse colors or
animations - create a "new school branding"

------------------------------------------------------------------------

## Design References

## Content Guidelines

### Tone of Voice

-   calm
-   clear
-   honest
-   neutral
-   non-persuasive

Always include messaging like: - "We are exploring..." - "No decision
has been made" - "We are carefully investigating..."

Avoid: - promotional language - definitive statements - overly
optimistic framing - jargon

------------------------------------------------------------------------

## UX Guidelines

Prioritize: - simplicity - clarity - predictable structure

Key pages: - Home - Explanation - Future Perspective - Timeline - FAQ -
Updates - Contact

Key elements: - video on homepage - FAQ - timeline - explanation for
parents

------------------------------------------------------------------------

## Components

-   Hero section
-   Video block
-   Content cards
-   FAQ accordion
-   Timeline
-   Updates
-   Contact

Use consistent spacing, rounded corners, soft shadows.

------------------------------------------------------------------------

## Logo Usage

-   Always equal
-   Same size and alignment
-   Prefer footer placement

Do NOT highlight one over the other.

------------------------------------------------------------------------

## Imagery

-   Real and authentic
-   Show collaboration
-   Avoid stock photos
-   Avoid bias

------------------------------------------------------------------------

## Motion

Allowed: - subtle fade - light hover

Avoid: - heavy animation - parallax

------------------------------------------------------------------------

## Accessibility

-   Minimum font size 18px
-   High contrast
-   Alt text
-   Keyboard navigation

------------------------------------------------------------------------

## Technical

-   Clean code
-   Reusable components
-   Responsive design
-   Use design tokens

------------------------------------------------------------------------

## Final Instruction

Choose clarity over creativity. Choose neutrality over visual impact.
Choose simplicity over complexity.

This is not a marketing website. It is a trust-based information
platform.
